I'm hoping you tried printing multiple lines when looking at the key questions for the last lesson. You probably tried something like this: 

	#include <stdio.h>
	
	int main(){
		printf("I am printing this statement ");
		printf("on two lines.");
		return(0);
	}
This however would output: 

	I am printing this statement on two lines.
What you need is to tell the program that the next statement is on a new line. You could technically write a program like this: 

	#include <stdio.h> intmain(){ printf("I am printing this statement "); printf("on two lines."); return(0); }
... but I wouldn't recommend it. To tell the program to print on a new line you can insert a \n at the end of your statement within the quotes. 

	#include <stdio.h>
	
	int main(){
		printf("I am printing this statement \n");
		printf("on two lines.");
		return(0);
	}
\n is acting like a character. \ is not printed to the screen and the combination of \n prints a new line. If you wanted to print quotes within a print statement, you would have to make sure the compiler new you were not ending the statement with those quotes. To do this you would type \". The output of the above program would now be: 

	I am printing this statement 
	on two lines.

To go through these lessons, you should probably have a basic understanding of Algebra. That will make this next part of the lesson much easier. In Algebra you have variables which stand for a specific value or any possible value that works in the equation they are in. Variable work exactly the same in programming. 

	int x = 10;
This statement is assigning the value 10 to the integer x.X can now be used later in the program. If you wanted to print the value of x, your entire program would look like this: 

	#include <stdio.h>
	
	int main(){
		int x = 10;
		printf("%d", x);
		return(0);
	}
Notice that there are no quotes around the variable x. Instead there is %d within the quotes. This tells the compiler to display a digit to the screen. After the end quote is placed in, it uses the variable to place in the digit. If we just printed x in quotes the output would be: 

	x
The output of our program is rather: 

	10
Variables can be modified just like in mathematics: 

	#include <stdio.h>
	
	int main(){
		int x = 5 + 3 * 7;
		printf("%d", x);
		return(0);
	}
Here the output would be 15. Order of operations is kept. Multiplication and division are calculated first and then addition and subtraction. If we wanted the calculation to be 5 + 3 and that * 7, we could change it by adding in parentheses. 

	#include <stdio.h>
	
	int main(){
		int x = (5 + 3) * 7;
		printf("%d", x);
		return(0);
	}
Now the output would be 56.

Variable names can be virutally anything but have some rules: 

	* Variable names should not be more than 31 characters in length (And this would also be a pain to type out each time. Keep it short!)
	
	* No commas, spaces or special symbols other than an underscore are allowed in the variable name.
	
	* The first character of the variable name must either be a letter or underscore. It should not start with the digit.
	
	* Variable names CAN CONTAIN capital letters.

What happens if you assign a different value to a variable later in a program? 

	#include <stdio.h>
	
	int main(){
		int x = 10;
		printf("%d \n", x);
		x = 6;
		printf("%d", x);
		return(0);
	}
Here 10 would be printed and then 6 would be printed on a new line. Notice I did not declare what variable type x was the second time. If I did, I would be calling two different variables with the same name. This is not allowed in ANSI-C (The type of the C language we are programming in.) If we put in the following code, 6 would be printed to the screen, because the value has been changed. The original value is not used: 

	#include <stdio.h>
	
	int main(){
		int x = 10;
		x = 6;
		printf("%d", x);
		return(0);
	}
There is much more we can do with variables, but we will revisit them in another section.
